# ReactJsonUI Development Guide

## Documentation Reference

When looking for information about JsonUI components, attributes, or features:
- **SwiftJsonUI Wiki**: `~/resource/SwiftyJsonUI_wiki/`

## Overview

ReactJsonUI is the React/Next.js implementation of the JsonUI cross-platform UI framework.

## Project Structure

```
ReactJsonUI/
├── rjui_tools/           # Code generation tools (Ruby)
│   ├── bin/              # Executable scripts
│   │   └── rjui          # Main CLI executable
│   └── lib/              # Core library code
└── example/              # Example Next.js application
```

## Important Rules

### Modifying rjui_tools
- **ALWAYS reference sjui_tools when modifying rjui_tools**
- SwiftJsonUI's sjui_tools is the reference implementation: `~/resource/SwiftJsonUI/tools/sjui_tools/`
- Ensure consistency with sjui_tools patterns and conventions
- Check sjui_tools for existing solutions before implementing new features

### Writing JSON Layout Files
- **NEVER guess or assume JSON attributes or component properties**
- If the information is not in the current context, **ALWAYS check the wiki first**:
  - `~/resource/SwiftyJsonUI_wiki/`
  - `~/resource/KotlinJsonUI_wiki/`
- Do not invent attribute names or values based on assumptions

### CLI Commands
```bash
# Initialize a new project
./rjui_tools/bin/rjui init

# Build generated files from JSON layouts
./rjui_tools/bin/rjui build

# Watch mode for development
./rjui_tools/bin/rjui watch

# Generate a new view
./rjui_tools/bin/rjui g view <view_name>

# Generate a new component
./rjui_tools/bin/rjui g component <component_name>

# Generate a new converter
./rjui_tools/bin/rjui g converter <converter_name>
```

### Auto-Generated Files
- **NEVER directly edit files in generated directories** - These are auto-generated by `rjui build`
- All changes should be made in:
  - JSON layout files
  - Style files
  - String files
  - ViewModel files

## Related Projects

JsonUI is a cross-platform UI framework that generates native views from the same JSON definition:

- **ReactJsonUI**: `~/resource/ReactJsonUI` - React/Next.js version (this project)
- **SwiftJsonUI**: `~/resource/SwiftJsonUI` - iOS/Swift version (UIKit & SwiftUI) - **Reference implementation**
- **KotlinJsonUI**: `~/resource/KotlinJsonUI` - Android/Kotlin version (XML & Compose)

All three projects share the same JSON schema, allowing a single JSON layout to render identical UI across web, iOS, and Android platforms.

## Reference Implementation

When implementing new features or fixing bugs in rjui_tools:
1. First check how sjui_tools handles it: `~/resource/SwiftJsonUI/tools/sjui_tools/`
2. Maintain consistency with the Swift implementation
3. Adapt patterns for React/TypeScript as needed
